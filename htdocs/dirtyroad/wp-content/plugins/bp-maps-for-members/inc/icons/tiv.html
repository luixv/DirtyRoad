GET/api/v2/accounts/{ucid}/vehicles
VehicleListWithContracts

Returns a (optionally filtered/sorted) list of all vehicles related to the given UCID, containing basic vehicle and contract information.

Parameters

NameDescription
language
string
(header)
ucid *
string
(path)

The unique customer id (UCID) as given from CCDB (Central Customer Database).

The UCID is the required first Path Parameter for every Vehicle API Resource.

page
integer
(query)

The number of the page to be returned. Pages are of a certain size (specified in parameter ‘size’).
Only positive numbers and 0 are allowed. Default is 0 - the first page.
If the requested page does not exist, an empty vehicle list is returned.

size
integer
(query)

The size of the page to be returned. Only positive numbers > 0 are allowed - default is 5.

filter
string
(query)

List of filter expressions like field:value(s) separated by ‘;’ - Filter expressions may be:

  • contract:uptime,fleetboard,… (list of downstream products uptime, fleetboard, finance, service, none)
  • classification:… (vehicle classification like active, unclassified, archive, ooi)
  • productline:… (product line, e.g. ACTROS, ATEGO, Unimog)

The filter expressions are combined via AND - the values of a single filter expression are interpreted with OR.
Downstream products may be: uptime (MB Uptime), fleetboard (Fleetboard Services), finance (Financial Services)service (Service Contracts), none(no downstream product available).

Example: filter=contract:uptime,fleetboard;classification:active

sort
string
(query)

A comma separated list of field name(s) to sort the results by multi-column sort.

The sort Order can be specified with an ending ‘+’/’-' for ascending/descending. If not specified, ascending order is assumed
Allowed field names are:

  • model
  • type
  • name
  • FIN
  • VIN
  • licencePlate
  • classificationDate
  • classificationType
  • classificationReason
  • firstRegistrationDate

Example: sort=type-,name+
